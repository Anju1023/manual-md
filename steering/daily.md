# 週に 2 回の作業

## 消耗品の発注

週に 2 回の発注

- 木曜 18 時までに発注 → 翌週水曜納品
- 火曜 18 時までに発注 → 土曜納品

1. `"C:\Users\nanbu\OneDrive\Desktop\発注書\エフピコ発注書.xlsx"`を開く
2. `J4セル`の数式を`A2セル`にコピー
3. `A2セル`をコピーし、そのまま値のみ貼り付け(日付)
4. 発注数を入力
5. 洗剤の発注は`洗剤発注書`シートへ入力
6. 本社の江藤さんと日下部さんにメールで送付

洗剤の発注もある場合はメールでひとこと伝える。
その他ボンジュールシールなどの発注も江藤さんへ

※`J4セル`の数式：

```excel
=LET(
    today,WEEKDAY(TODAY(),2),
    nextSat,TODAY()+7-WEEKDAY(TODAY(),1)+IF(WEEKDAY(TODAY(),1)<7,0,7),
    nextWed,TODAY()+7-WEEKDAY(TODAY(),1)+IF(WEEKDAY(TODAY(),1)<4,-3,4),
    "発注日:  "&TEXT(TODAY(),"yyyy年  mm月  dd日(aaa)")&"⇒納品日:   "&TEXT(IF(today<=2,nextSat,nextWed),"  mm月  dd日(aaa)")
)
```

# 月末の作業

## 棚卸 → 清書

**記入前の手順：**

1. `"C:\Users\nanbu\OneDrive\Desktop\センター棚卸表\"`の先月分の棚卸表をコピー
2. ファイル名を今月に書き換える
3. ファイルを開く
4. `I1セル`を今月末に書き換える
5. 当月在庫数の欄を前月在庫数にコピーし、当月在庫数の欄は空白にする
6. 印刷する

**記入後の手順：**

1. 在庫数を記入してもらったら、先ほどのファイルに当月在庫数を入力する
2. できれば翌月 1 日までに本社の伊藤部長にメールで送付する

## シフト入力 → 清書

**記入前の手順：**

1. `"C:\Users\nanbu\OneDrive\Desktop\よく使う\シフト表.xlsx"`を開く
2. 今月分のシートを複製し、シート名を翌月へ
3. エラーが出た場合、同じシート名が存在するので、去年のシートは削除する
4. 日付が 11 日~10 日になっているか確認し、なっていなければ列の挿入・削除を行う
5. 今月の年休残数をコピーし、翌月分の`AL5セル`から下に値のみ貼り付け
6. 今月分のシフトが残っているのですべて削除
7. シフトがすべて空白になった状態で印刷

**記入後の手順：**

1. 部長からシフト表を受け取り、同じように入力する
2. 半休の場合は`半休`と入力しないと反映されないので注意
3. 入力が終わったら印刷

パートさんに LINE で送る場合は、PDF にエクスポートし、変換サイトで PNG に変換し、周知

※休日の列の数式：

```excel
=SUM(
    COUNTIF(C5:AG5,"×"),
    COUNTIF(C5:AG5,"年"),
    COUNTIF(C5:AG5,"／"),
    COUNTIF(C5:AG5,"健診"),
    COUNTIF(C5:AG5,"*半*")*0.5,
    COUNTIF(C5:AG5,"AM")*0.5,
    COUNTIF(C5:AG5,"PM")*0.5
)
```

# 随時行う作業

## 新商品処理

### レシピの用意

CB 新規登録依頼に必要なレシピを用意する。

- 商品名
- 売価
- 材料
- ホイロ・窯の温度
- 工程
- 特徴
- 商品画像

を入力し、
`"C:\Users\nanbu\Dropbox\共有\レシピ\新ボンジュール\商品レシピ\"`に保存しておく

### CB 新規登録依頼

1. `"C:\Users\nanbu\OneDrive\Desktop\新規登録\CB新規登録申請書.xlsx"`を開く
2. `Sheet2`の`B2セル`の日付を確認し、その日付を`新規登録申請書`シートの POS 発行日に入力
3. 商品名、レシート商品名、売価を入力(その他はそのまま)
4. 本社の江藤さんにこのファイルと、レシピをメールで送付し、登録を依頼

### @Label に登録

1. 江藤さんから JAN コードが送られてくるので、控えておく
2. `"C:\Users\nanbu\Dropbox\共有\TERAOKAラベラー\ラベルプリンター　リスト\旧コンパクトベーカリー\コンパクトベーカリー商品リスト.xlsx"`を開く
3. カテゴリに沿ったシートの空いている呼び出し番号を選ぶ
4. 商品名、売価、原材料名、消費期限、JAN コード、呼び出し番号を入力
5. [@Label](https://labelserv.net/)を開く
6. ログイン ID とパスワードを入力し、ログイン
7. `商品マスタ`タブの`商品マスタ`をクリック
8. 先ほど決めた呼び出し番号を入力し、検索
9. 存在していれば、一番上に出てくるので、選択した状態で変更ボタンをクリック
10. 以下の項目を変更
    - 配信設定 → 配信する
    - 品名 1 行目 → 商品名 (半角は使えない)
    - バー区分 → 08...
    - JAN コード → JAN コード
    - 計量区分/単価 → 1：定額 売価
    - 賞味期限 → 賞味期限
    - 添加物コード → 呼び出し番号と同じ ※1
    - 保存温度コード → 000001
11. 登録ボタンをクリック
12. `メンテナンス`タブの`今すぐマスタ配信`をクリック
13. 店舗を選択し、配信するボタンをクリック

※1 添加物コードは変更ボタンを押して、一括入力
原材料名を貼り付け、繋げて改行
決定 → 登録

### CB 商品データに登録

1. `"C:\Users\nanbu\OneDrive\Desktop\よく使う\商品データ\CB商品データ.xlsm"`を開く
2. `操作`シートの下の方に新規入力ボタンがあるのでクリック
3. カテゴリを選択すると商品 ID が生成される
4. 売価など必要項目を入力し、登録
5. `終売`にチェックを入れていなければ、各店舗の数量シートに反映される
6. 数量が空白だとエラーが起きるので、一旦すべて`0`にする

### 納品記録用データに登録

1. `"C:\Users\nanbu\OneDrive\Desktop\よく使う\伝票作成\T納品データ.xlsm"`を開く
2. `T商品マスタ`シートの商品 ID、商品名、終売を入力
3. `T価格履歴マスター`シートの商品 ID、社名、適用開始日、売価を入力

### 数量の入力

1. 社長に数量を聞く
2. 返ってきた数量表をもとに各店舗数量シートに入力する
3. `操作`シートの全部まとめて出力ボタンをクリック
4. 製造数など更新された表を印刷

### 伝票用シートに欄を追加

1. `伝票用`シート(クリエイト分も)の商品列の一番下の行の下に行を挿入
2. 商品名、売価等を記入

## 終売処理

1. 終売になる商品が 2 週間分すべて 0 になったら実行
2. 商品データの終売に〇をつける
3. 各店舗の数量シートでズレた行を削除して調整

## 数量変更依頼

1. 本社から 1~3 店舗ほど変更依頼が送られてくる
2. 送られてきた資料をもとに各店舗数量シートを変更
3. `操作`シートの全部まとめて出力ボタンをクリック
4. 製造数など、変更のあった表を印刷・掲示
